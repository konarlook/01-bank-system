# Parslib

Проект по модулю 1. Библиотека `parslib` для чтения, записи, конвертации и
сравнения финансовых операций в трех форматах (`csv`, `txt` и `bin`) и два
консольных приложения на нее основе:

* **converter** (`str/main.rs`) - конвертирует файл с операциями из одного
  формата в другой;
* **comparer** (`src/bin/comparer.rs`) - сравнивает два файла после парсинга
  в общее представление и показывает первое расхождение транзакций;

Оба приложения используют публичный API библиотеки: вся логика парсинга,
записи и сравнения транзакций находится в `parslib`, ввод и вывод
работают через абстракции `std::io::Read`/`std::io:Write`.

## Сборка и проверка

```sh
cargo build
cargo fmt
cargo check
cargo test
```

## converter

Конвертирует операции между форматами. Формы входа и выхода определяются
по расширению файлов: `.csv`, `.txt`, `.bin`.

```sh
cargo run --bin bank-system -- <входной-файл> <выходной файл>
```

Примеры (тестовые файлы лежат в `example/`):

```sh
cargo run --bin bank-system -- example/test.csv out.txt
cargo run --bin bank-system -- example/test.txt out.bin
cargo run --bin bank-system -- example/test.bin out.csv
```

При неизвестном расширении, отсутствующем файле или некорректных данных
программа печатает ошибку в stderr и завершается с кодом 1.

## comparer

Сравнивает два файла с операциями (форматы могут быть разными - сравнение
идет после парсинга, а не по байтам).

```sh
cargo run --bin comparer -- <первый-файл> <второй-файл>
```

Пример:

```sh
cargo run --bin comparer -- example/test.csv example/test.txt
```

### bin

Бинарный формат, все числа — **little-endian**, строки — UTF-8.
Заголовок файла:

| Смещение | Размер | Содержимое                   |
|----------|--------|------------------------------|
| 0        | 4      | магические байты `TRNS`      |
| 4        | 4      | `u32` — количество записей N |

Далее подряд N записей, каждая:

| Поле       | Размер | Содержимое                             |
|------------|--------|----------------------------------------|
| длина даты | 4      | `u32` — длина строки даты в байтах (L) |
| дата       | L      | строка даты, UTF-8                     |
| категория  | 1      | `u8`: `0` = salary, `1` = rent         |
| тип        | 1      | `u8`: `0` = income, `1` = expense      |
| сумма      | 8      | `i64` — сумма                          |

Пример: файл с одной операцией `2026-04-01;salary;income;120000`
занимает 32 байта: `TRNS` + count=1 + len=10 + `2026-04-01` + `00` + `00` + сумма.

Файл с неверными магическими байтами, неизвестным кодом категории/типа
или обрезанной записью даёт ошибку чтения.
